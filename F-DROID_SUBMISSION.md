# F-Droid Submission Guide for HabitFlow

## What is F-Droid?
F-Droid is a free and open-source app repository for Android. It's an excellent platform for privacy-focused, open-source apps like HabitFlow.

## Prerequisites
- ✅ Open-source code (MIT License - already included)
- ✅ Privacy-focused (no tracking, no ads)
- ✅ No proprietary dependencies
- ✅ Source code on publicly accessible repository (GitHub)

## Steps to Submit to F-Droid

### 1. Prepare Your Repository

Choose either option:

**Option A: GitHub (Recommended)**
```bash
# Create a new GitHub repo
git init
git add .
git commit -m "Initial commit: HabitFlow v1.0.0"
git remote add origin https://github.com/YOUR_USERNAME/HabitFlow.git
git branch -M main
git push -u origin main
```

**Option B: GitLab**
- Similar process, use your GitLab account URL

### 2. Create Release on GitHub

```bash
# Create a Git tag for the version
git tag -a v1.0.0 -m "HabitFlow v1.0.0 - First Release"
git push origin v1.0.0

# Or use GitHub UI to create a release with:
# - Tag: v1.0.0
# - Title: HabitFlow v1.0.0
# - Built APK file attached
```

### 3. Build Release APK

```bash
# Navigate to project directory
cd HabitFlow

# Get latest dependencies
flutter pub get

# Build release APK
flutter build apk --release

# APK location: build/app/outputs/flutter-apk/app-release.apk

# (Optional) Build App Bundle
flutter build appbundle --release
# AAB location: build/app/outputs/bundle/release/app-release.aab
```

### 4. Submit to F-Droid

F-Droid uses an automated build system:

1. **Create a Metadata File** (`fastlane/metadata/android/en-US/`)
   - ✅ Already done with `title.txt`, `short_description.txt`, `full_description.txt`

2. **Submit to F-Droid Data Repository**
   - Fork: https://gitlab.com/fdroid/fdroiddata
   - Create new app metadata in `metadata/com.habitflow.app.yml`

**Example F-Droid Metadata File:**

Create `metadata/com.habitflow.app.yml` in fdroiddata repo:

```yaml
Categories:
  - System
License: MIT
AuthorName: Your Name
AuthorEmail: your.email@example.com
SourceCode: https://github.com/YOUR_USERNAME/HabitFlow
IssueTracker: https://github.com/YOUR_USERNAME/HabitFlow/issues
Changelog: https://github.com/YOUR_USERNAME/HabitFlow/releases
Donate: https://github.com/YOUR_USERNAME/HabitFlow

Builds:
  - versionName: '1.0.0'
    versionCode: 1
    commit: v1.0.0
    output: app-release.apk
    build:
      - echo 'Building HabitFlow...'
    ndk: r23c

AllowedAPKSigningKeys: # Will be generated by F-Droid
  - EXAMPLEFINGERPRINT

CurrentVersionCode: 1
```

3. **Create a Pull Request** to fdroiddata repo
   - F-Droid maintainers will review your submission
   - They'll verify the source code and build system
   - Typical review time: 1-4 weeks

### 5. Maintain Your App

Once approved:
- Updates are automatic when you push Git tags
- F-Droid rebuilds your app from source
- Changelog is automatically updated
- No manual APK uploads needed

## App Signing

F-Droid handles APK signing automatically. However, you should create a keystore for development:

```bash
# Create keystore for local builds (optional)
keytool -genkey -v -keystore ~/release-key.keystore \
  -keyalg RSA -keysize 2048 -validity 10000 \
  -alias habitflow-release

# Sign APK manually if needed
jarsigner -verbose -sigalg SHA256withRSA \
  -digestalg SHA-256 \
  -keystore ~/release-key.keystore \
  app-release-unsigned.apk habitflow-release
```
